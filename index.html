Sistema de Conversao de Cartao Ponto

Documentacao em formato de wiki para o entendimento do software.

MIT Licence - Baseada na versao 1.0.0 do software
Arquivo read.py
read_hours_one(quantity_of_pages: int, path: string, destiny_path:string)

quantity_of_pages: valor dado na interface, controle do usuário;
path: valor dado na interface. Arquivo de origem para conversão, controle do usuário;
destiny_path: Pasta de destino do arquivo final dado pelo usuário.
Leitura das páginas PDF são feitas dentro do loop while.
Lê o modelo antigo dos cartões ponto. A cada página lida, e criado um arquivo csv com o resultado da leitura feita pelo tabula. Após a criação do arquivo, essas informações são passadas para um arquivo chamado all_pages.csv já com os filtros feitos – filtros do arquivo filters.py.
Logo após os dados filtrados entrarem em all_pages.csv, o arquivo original e excluído. Para cada página e feito esse processo.
